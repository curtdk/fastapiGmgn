# 交易簇组（Cluster）完整模块设计方案
## 一、概述
基于交易解析函数`_extract_trade_info`提取的完整指令、程序、计算单元特征，构建可配置、可开关、自动聚类、可人工锁定、可展示联动的簇组系统，用于批量识别、标记、管理散户/庄家交易，实现特征聚类 + 状态判定 + 人工终审的完整流程。

---

## 二、可配置匹配条件（支持独立开关 + 区间偏移）
所有条件与其他判断条件并行同级，簇组功能整体支持总开关；已开启的条件必须全部满足才算匹配成功：
1. CU（计算单元）：基准值 ± 偏移量
2. 程序ID数量：基准数 ± 偏移量
3. 主指令数量：基准数 ± 偏移量
4. 内部指令数量：基准数 ± 偏移量

---

## 三、簇组结构（Redis Key = 簇组名称）
### 1. 标识信息
- 簇组名称（首个钱包地址）
- 所属文件夹（默认为空）

### 2. 状态信息
- 是否启用
- 簇组类型：未定义 / 散户 / 庄家
- 判定类型：系统判定 / 手动锁定

### 3. 基准特征（完整比对信息）
- 基准CU
- 基准程序ID数量、程序ID完整列表与地址
- 基准主指令数量、主指令名称与地址
- 基准内部指令数量、内部指令名称与地址

### 4. 成员数据（自动去重）
- 所属Tx列表
- 所属用户（钱包地址）列表

### 5. 统计数据
- 总Tx数
- 总独立用户数

---

## 四、系统自动判定 + 手动锁定规则（合并版）
1. 可配置阈值：自定义Tx数阈值、用户数阈值（默认均＞50）
2. 系统判定：未手动修改的簇组，满足双阈值 → 自动标记为「庄家」
3. 手动锁定：人工修改簇组类型（散户/庄家）后，系统永久不再自动覆盖
4. 优先级：手动锁定 ＞ 系统判定

---

## 五、Tx 与用户关联逻辑
1. 每笔Tx匹配簇组后，绑定所属簇组完整信息
2. 每个用户（钱包）自动关联所属簇组
3. 实时交易列表展示：
   - 簇组缩写、ID、名称
   - 簇组状态（未定义/散户/庄家）
4. 点击列表中簇组信息，可直接修改状态并触发锁定

---

## 六、运行时处理流程
1. 新Tx进入，先判断簇组总开关是否开启
2. 遍历所有已启用簇组，按开启的特征条件+区间匹配
3. 匹配成功：
   - Tx加入簇组
   - 钱包地址去重加入用户列表
   - 更新统计数
   - Tx与用户继承簇组状态
4. 无匹配：自动新建簇组，以首个钱包地址命名
5. 数据更新后：未锁定簇组按阈值自动判定类型

---

## 七、簇组管理页面功能
1. 列表展示：簇组信息、状态、启用开关、统计数、判定类型
2. 查看簇组内所有Tx详情（同实时交易页面）
3. 支持操作：
   - 启用/禁用
   - 修改所属文件夹
   - 修改簇组类型（手动锁定）
4. 手动修改后永久锁定，系统不再变更
