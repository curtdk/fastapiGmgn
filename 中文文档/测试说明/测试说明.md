# 交易监控系统 - 测试说明

## 启动服务

```bash
cd /Users/curtdk/.openclaw/workspace/fastapiGmgn
source venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

## 访问地址

- 管理后台: http://localhost:8000/admin
- 默认账号: admin / admin123
- 登录后默认跳转到交易监控页面

## 功能测试步骤

### 1. 测试登录

1. 打开浏览器访问 http://localhost:8000/admin
2. 使用 admin / admin123 登录
3. 确认登录后进入"交易监控"页面

### 2. 测试交易监控页面

1. 在 Mint 输入框中输入一个 Solana Token 的 Mint 地址
   - 测试用地址示例: `DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263` (Bonk)
2. 点击"开始"按钮
3. 观察状态变化: IDLE → BACKFILLING → CATCHING_UP → STREAMING
4. 观察交易表格是否开始填充数据
5. 观察四个指标是否更新

### 3. 测试停止监听

1. 点击"关闭"按钮
2. 确认状态变为 STOPPED
3. 确认不再有新数据进入

### 4. 测试系统设置

1. 在左侧菜单点击"系统设置"
2. 修改参数（如 batch_size 改为 50）
3. 点击"保存全部"
4. 确认保存成功

### 5. 测试 WebSocket 实时推送

1. 启动监听后，打开浏览器开发者工具 → Network → WS
2. 确认 WebSocket 连接成功
3. 观察收到的消息类型: status, trade, metrics

## API 测试（curl）

### 启动监听
```bash
curl -X POST http://localhost:8000/admin/api/trades/DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263/start
```

### 获取交易列表
```bash
curl http://localhost:8000/admin/api/trades/DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263?page=1&page_size=10
```

### 获取指标
```bash
curl http://localhost:8000/admin/api/trades/DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263/metrics
```

### 停止监听
```bash
curl -X POST http://localhost:8000/admin/api/trades/DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263/stop
```

### 获取设置
```bash
curl http://localhost:8000/admin/api/settings
```

### 更新设置
```bash
curl -X PUT http://localhost:8000/admin/api/settings/batch_size \
  -H "Content-Type: application/json" \
  -d '{"value": "50"}'
```

## 数据库结构

### 新增表

**settings** - 系统设置表
| 字段 | 类型 | 说明 |
|------|------|------|
| key | TEXT (PK) | 设置键名 |
| value | TEXT | 设置值 |
| description | TEXT | 描述 |
| updated_at | DATETIME | 更新时间 |

**trade_analysis** - 交易分析结果表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER (PK) | 自增ID |
| sig | TEXT (UNIQUE) | 交易签名 |
| net_sol_flow | REAL | SOL净流入 |
| net_token_flow | REAL | Token净流入 |
| price_per_token | REAL | 单价(SOL) |
| wallet_tag | TEXT | 钱包标记 |
| processed_at | DATETIME | 处理时间 |
