# 用户状态判断逻辑

## 状态定义

| 状态 | 含义 | 初始值 |
|------|------|--------|
| `unknown` | 不确定 | ✓ 默认 |
| `dealer` | 庄家 | - |
| `retail` | 散户 | - |

## _default_trader_state 简化

新用户默认状态只包含 status 和 conditions：
```python
def _default_trader_state() -> dict:
    """默认交易员状态（只保留 status 和 conditions，持仓数据从 Redis 读取）"""
    return {
        "status": "unknown",
        "conditions": [],
    }
```

**原因**：持仓数据（holdingQty、holdingCost 等）统一从 Redis 的 `{mint}_xxx` key 读取，不需要在默认值中定义。

## 判断流程

```
新用户 → unknown（默认）
   │
   ├─→ C002-C005 满足？ → dealer（get_trader_state 第73行）
   │                       直接判定，跳过 C001
   │
   └─→ 入队等待 C001
           │
           └─→ _run_dealer_check（C001 检测）
                   │
                   └─→ _apply_dealer_result
                           │
                           ├─→ is_dealer=True  → dealer
                           │
                           └─→ is_dealer=False → retail
```

## 判断点说明

### 判断庄家（2 个地方）

1. **get_trader_state**（C002-C005）
   - 调用 `_check_local_dealer_conditions` 快速判断
   - 无需远程 API
   - 满足条件直接标记为 dealer

2. **_apply_dealer_result**（C001）
   - 通过 Helius API 检测首笔交易
   - 满足 C001 标记为 dealer

### 判断散户（1 个地方）

- **_apply_dealer_result**（第250行）
  ```python
  state["status"] = "dealer" if is_dealer else "retail"
  ```

**注意**：`_check_local_dealer_conditions` 只能判断庄家，散户只在 C001 检测完成后才能确定。

## 已知问题

### 重试失败后状态仍为 unknown

当 C001 检测重试 3 次都失败后（第129-142行），用户状态会保持 `unknown`，不会变成 `retail`：

```python
if new_count >= 3:
    # 重试 3 次都失败，广播 unknown
    await ws_manager.broadcast(mint, {
        "type": "user_status",
        "data": {
            "status": "unknown",  # ← 仍然是 unknown！
            "reason": "detection_failed",
            ...
        }
    })
```

**影响**：这些用户会一直被当作 unknown 状态处理，不会参与指数计算但也不会被明确标记为散户。
