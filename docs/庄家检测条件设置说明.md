# 庄家检测条件设置说明

## 概述

庄家检测功能通过多个条件（C001-C005）综合判断一个钱包地址是否可能是庄家。每个条件都可以独立启用或禁用，并支持配置阈值参数。

## 设置页面

访问地址：`/admin/dealer-settings`

## 条件配置列表

| 条件 | 启用开关 | 阈值配置 | 默认值 | 说明 |
|------|---------|---------|--------|------|
| C001 | `dealer_c001_enabled` | - | false | 首笔交易 closeAccount |
| C002 | `dealer_alt_enabled` | - | false | 使用 ALT（地址查找表） |
| C003 | `dealer_gas_enabled` | `dealer_gas_max` | false / 0.00001 | Gas 费小于阈值 |
| C004 | `dealer_cu_enabled` | `dealer_cu_min`, `dealer_cu_max` | false / 0, 200000 | CU 消耗在范围内 |
| C005 | `dealer_risk_enabled` | `dealer_risk_min` | false / 0 | 风险分大于阈值 |

## 详细配置

### C001: 首笔交易 closeAccount

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dealer_c001_enabled` | bool | false | 是否启用该条件 |

**检测逻辑**：
- 获取钱包最旧一笔交易（通过 Helius API）
- 检查是否包含 `closeAccount` 指令
- 检查交易后该地址 SOL 余额是否为 0

**代码位置**：
- 配置定义：`app/services/settings_service.py`
- 检测逻辑：`app/services/dealer_detector.py` -> `_run_dealer_check()` -> `if c001_enabled == "true":`
- API 调用：`app/services/dealer_detector.py` -> `_fetch_first_transaction()`
- 检测函数：`app/services/dealer_detector.py` -> `_check_c001()`

---

### C002: ALT 条件

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dealer_alt_enabled` | bool | false | 是否启用该条件 |

**检测逻辑**：
- 检查交易的 `uses_lookup_table` 字段是否为 True
- 使用了地址查找表（Address Lookup Table）的钱包被认为是庄家

**代码位置**：
- 配置定义：`app/services/settings_service.py`
- 检测逻辑：`app/services/dealer_detector.py` -> `_run_dealer_check()` -> C002 块

---

### C003: Gas 费条件

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dealer_gas_enabled` | bool | false | 是否启用该条件 |
| `dealer_gas_max` | float | 0.00001 | Gas 费最大值 (SOL) |

**检测逻辑**：
- 检查交易的手续费是否小于 `dealer_gas_max` 阈值
- Gas 费很低的钱包被认为是庄家

**代码位置**：
- 配置定义：`app/services/settings_service.py`
- 检测逻辑：`app/services/dealer_detector.py` -> `_run_dealer_check()` -> C003 块

---

### C004: CU 条件

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dealer_cu_enabled` | bool | false | 是否启用该条件 |
| `dealer_cu_min` | int | 0 | CU 消耗最小值 |
| `dealer_cu_max` | int | 200000 | CU 消耗最大值 |

**检测逻辑**：
- 检查交易的 `cu_consumed` 是否在 `[cu_min, cu_max]` 范围内
- CU 消耗稳定的钱包被认为是庄家

**代码位置**：
- 配置定义：`app/services/settings_service.py`
- 检测逻辑：`app/services/dealer_detector.py` -> `_run_dealer_check()` -> C004 块

---

### C005: 风险分条件

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dealer_risk_enabled` | bool | false | 是否启用该条件 |
| `dealer_risk_min` | int | 0 | 风险分最小值 |

**检测逻辑**：
- 检查交易的 `risk_score` 是否大于 `dealer_risk_min` 阈值
- 风险分高的钱包被认为是庄家

**代码位置**：
- 配置定义：`app/services/settings_service.py`
- 检测逻辑：`app/services/dealer_detector.py` -> `_run_dealer_check()` -> C005 块

## 检测执行流程

```
用户交易触发 → 指数计算 → 状态为 unknown → 入队庄家检测
                                              ↓
                                    _dealer_queue_consumer()
                                              ↓
                                    _run_dealer_check(address, mint, sig)
                                              ↓
                    ┌───────────────────────────────────────────────┐
                    ↓                   ↓                   ↓         ↓
                  C002 检查          C003 检查          C004 检查   C005 检查
                    ↓                   ↓                   ↓         ↓
              满足条件?           满足条件?           满足条件?    满足条件?
                    ↓                   ↓                   ↓         ↓
                    └───────────────────┴───────────────────┴─────────┘
                                              ↓
                                    任意条件满足 → dealer
                                              ↓
                              无条件满足 → C001 检查（Helius API）
                                              ↓
                                    满足 C001 → dealer
                                              ↓
                                    不满足 → retail
```

## 数据库存储

设置项存储在 `Setting` 表中：
- `key`: 配置项名称
- `value`: 配置值（字符串）
- `description`: 描述信息

## Redis 数据结构

用户状态存储在 `user:{address}`：
- `status`: "unknown" | "dealer" | "retail"
- `conditions`: '["C001", "C002", ...]'
- `{mint}_holdingQty`: 持仓数量
- `{mint}_holdingCost`: 持仓成本
- 其他持仓相关字段

## API 接口

### 获取所有设置
```
GET /admin/api/settings
Response: { "dealer_c001_enabled": "false", ... }
```

### 更新单个设置
```
PUT /admin/api/settings
Body: { "key": "dealer_gas_max", "value": "0.00002" }
Response: { "key": "dealer_gas_max", "value": "0.00002" }
```

- 支持启用/禁用复选框
- 支持阈值输入
