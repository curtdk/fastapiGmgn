# 网页 3 个指数计算逻辑说明

## 一、指数定义

网页上显示的 3 个核心指数：

| 网页显示名称 | API 字段 | Redis 字段 | 含义 |
|--------------|----------|------------|------|
| **本轮下注 (SOL)** | `current_bet` | `total_bet` | 市场总锁仓深度（全池总投入） |
| **本轮成本 (SOL)** | `current_cost` | `total_bet - realized_profit` | 未变现的持仓价值 |
| **已落袋 (SOL)** | `realized_profit` | `realized_profit` | 所有用户落袋的收益总和 |

## 二、数据结构

### 2.1 全局指标（Redis metrics:{mint}）

| 字段 | 初始值 | 说明 |
|------|--------|------|
| `total_bet` | 0 | 全池总投入（所有用户持仓成本之和） |
| `realized_profit` | 0 | 全池已实现总收益 |
| `dealer_count` | 0 | 庄家数量 |

### 2.2 用户状态（Redis user:{address}）

| 字段 | 初始值 | 说明 |
|------|--------|------|
| `holdingQty` | 0 | 当前持有代币数量 |
| `holdingCost` | 0 | 当前持仓成本（SOL） |
| `avgPrice` | 0 | 持仓均价 = holdingCost / holdingQty |
| `totalBuyAmount` | 0 | 累计买入总金额 |
| `totalSellAmount` | 0 | 累计卖出总金额 |
| `totalSellPrincipal` | 0 | 累计卖出本金 |

## 三、均价法计算逻辑

### 3.1 买入交易

```
买入金额 = sol_spent（绝对值）
买入数量 = amount

holdingQty += buy_qty
holdingCost += buy_amount
totalBuyAmount += buy_amount
avgPrice = holdingCost / holdingQty
```

### 3.2 卖出交易

```
卖出数量 = amount（绝对值）
卖出金额 = sol_spent（绝对值）
卖出本金 = sell_qty × avgPrice  ← 关键：用当前持仓均价计算

totalSellAmount += sell_amount
totalSellPrincipal += sell_principal
holdingQty -= sell_qty
holdingCost -= sell_principal
avgPrice = holdingCost / holdingQty（如果还有持仓）

落袋收益 = sell_amount - sell_principal
```

### 3.3 全局指标更新

```
old_holding_cost = 交易前 holdingCost
new_holding_cost = 交易后 holdingCost
old_realized = totalSellAmount - totalSellPrincipal（交易前累计）
new_realized = totalSellAmount - totalSellPrincipal（交易后累计）

delta_bet = new_holding_cost - old_holding_cost
delta_profit = new_realized - old_realized

total_bet += delta_bet
realized_profit += delta_profit
```

## 四、完整示例

### 场景：用户进行 3 轮交易

**初始状态：** 0 Token，0 SOL

#### 第 1 步：买入 100 Token，花费 1 SOL

```
用户状态：
  holdingQty = 100
  holdingCost = 1.0 SOL
  avgPrice = 0.01 SOL/Token

全局指标：
  total_bet += 1.0  →  total_bet = 1.0
  realized_profit += 0  →  realized_profit = 0
```

#### 第 2 步：再买入 100 Token，花费 3 SOL

```
用户状态：
  holdingQty = 200
  holdingCost = 4.0 SOL
  avgPrice = 4.0 / 200 = 0.02 SOL/Token

全局指标：
  total_bet += 3.0  →  total_bet = 4.0
  realized_profit += 0  →  realized_profit = 0
```

#### 第 3 步：卖出 100 Token，获得 5 SOL

```
卖出本金 = 100 × 0.02 = 2.0 SOL
落袋收益 = 5.0 - 2.0 = 3.0 SOL

用户状态：
  holdingQty = 100
  holdingCost = 4.0 - 2.0 = 2.0 SOL
  avgPrice = 0.02 SOL/Token

全局指标：
  total_bet -= 2.0  →  total_bet = 2.0
  realized_profit += 3.0  →  realized_profit = 3.0
```

### 最终指数

| 指数 | 值 | 说明 |
|------|-----|------|
| 全池总投入 | 2.0 SOL | 剩余持仓成本 |
| 已实现收益 | 3.0 SOL | 所有用户落袋 |
| 当前持仓成本 | -1.0 SOL | 2.0 - 3.0 = -1（亏损状态） |

## 五、关键概念：卖出本金

卖出本金是"相当于最初投入了多少 SOL 买的这批币"，用持仓均价计算：

```
卖出本金 = 卖出数量 × 当前持仓均价
```

**为什么用均价计算？**
- 均价反映了用户的平均买入成本
- 卖出时，先卖出先买的部分，本金按当时的均价计算
- 实际实现了一个"先进先出"的成本计算逻辑

**落袋收益计算：**
```
落袋收益 = 卖出收入 - 卖出本金
```

如果落袋收益 > 0，说明这次卖出赚钱了；如果 < 0，则亏钱。

## 七、指数计算时机

1. **实时流交易**：每收到一笔新交易，立即计算更新
2. **回填完成**：触发全量计算（从旧到新顺序）
3. **手动刷新**：调用 calculate_metrics API

## 八、注意事项

1. **庄家排除**：庄家的交易不参与汇总计算
2. **防负值**：holdingQty < 0.001 时重置为 0
3. **精度处理**：所有计算使用 float，最后转为字符串存储
