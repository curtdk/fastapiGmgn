## 从测试数据得出的判断条件

### 测试发现

通过分析 4 个庄家的交易数据，发现以下特征：

| 钱包 | 第一笔交易 CU | 交易类型 | closeAccount |

| 4J2VVcXrFQi2y8woNA2SXi4a3NHZvX3irNs5H7LJUqwF | **109,289** | Sell | 需要 logs 检测 |

### API 限制

Helius API 返回 `logs = 0`（空），无法直接检测 `closeAccount`。

但 `computeUnitsConsumed` 值是准确的。

## 判断条件

### 核心条件：C003（CU 值）

```python
def check_c003_cu_range(info: dict) -> bool:
    """庄家 Sell 交易特征：CU 在 105,000-115,000"""
    cu = info.get("cu_consumed", 0)
    return 105000 <= cu <= 115000
```

**依据**：庄家 Sell 交易的 CU 值稳定在 ~109,000

### 辅助条件

| 条件 | 描述 | 依据 |
|------|------|------|
| C005 | Token Balance 大幅减少 | 庄家卖出代币 |
| C006 | 同一天内多笔交易 | 子钱包频繁操作 |

## 判断逻辑

```python
def is_dealer(info: dict, txs: list) -> tuple:
    """判断是否为庄家"""
    conditions = []
    
    # C003: CU 范围
    if check_c003_cu_range(info):
        conditions.append("C003")
    
    # C005: Token 大量减少
    if check_c005_token_dump(info):
        conditions.append("C005")
    
    # 庄家判定：C003 是核心条件
    is_dealer = check_c003_cu_range(info)
    
    return is_dealer, conditions
```

---

## 总结

**主要判断条件：C003（Compute Units 在 105,000-115,000）**

原因：
1. 庄家 Sell 交易的 CU 值稳定在 ~109,000
2. Helius API 不返回 logs，但返回准确的 CU 值
3.散户交易的 CU 值通常较低（< 10,000）
