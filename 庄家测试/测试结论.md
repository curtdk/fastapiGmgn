# 庄家检测测试结论

## 测试结果

### 测试汇总

| 类型 | 正确识别 | 准确率 |
|------|---------|--------|
| 庄家钱包 | 11/16 | **68.8%** |
| 散户钱包 | 5/11 | 45.5% |

### 关键发现

#### 庄家钱包 CU 特征
- 范围: 450 - 137,971
- Sell 交易 CU: **106,000 - 115,000**
- 11 个钱包命中 >105k CU

#### 散户钱包 CU 特征
- 范围: 3,609 - 480,809
- 10 个散户也有 >105k 的 CU
- 散户 CU 分布更广（包含超高值如 400k+）

### 未识别原因

5 个庄家钱包未检测到：
- F7Lyr3ho... (最大 CU: 3,300)
- 6EssiA1o... (最大 CU: 3,300)
- 91zFBYzxy... (最大 CU: 450)
- 8GmU23kLS... (最大 CU: 450)
- 4J2VVcXr... (最大 CU: 450)

**原因**：这些钱包只做了充值操作，没有 Sell 操作

### 误判原因

6 个散户被误判为庄家：
- 这些散户可能使用了其他交易机器人
- 或参与了较大金额的交易

## 优化建议

### 方案 1：扩大 CU 范围
```python
# 庄家 Sell 交易特征
庄家 CU: 106,000 - 115,000  # 高置信度
散户 CU: 120,000 - 480,000   # 超高 CU 不像庄家
```

### 方案 2：结合 C006（多笔交易）
```python
# 庄家通常有多笔 Sell 交易
if c003_count >= 2:  # 多于 1 笔 >100k CU
    is_dealer = True
```

### 方案 3：检测 SOL 变化模式
```python
# 庄家 Sell 后 SOL 增加
if sol_change > 0 and 100000 <= cu <= 115000:
    is_dealer = True
```

## 最终建议

**C003 条件需要优化为**：

```python
def check_c003_optimized(tx_data: dict) -> dict:
    """优化后的 C003 检测"""
    meta = tx_data.get("meta", {})
    cu = meta.get("computeUnitsConsumed", 0)
    
    # 庄家 Sell 交易特征：CU 在 106,000-115,000 且 SOL 增加
    sol_change = calculate_sol_change(meta)
    
    return {
        "cu": cu,
        "in_range": 106000 <= cu <= 115000,
        "is_sell_pattern": sol_change > 0
    }

def is_dealer(wallet_txs: list) -> bool:
    """综合判断"""
    c003_txs = [tx for tx in wallet_txs if check_c003_optimized(tx)["in_range"]]
    
    # 庄家特征：多笔 Sell 交易（>1 笔）
    return len(c003_txs) >= 2
```
